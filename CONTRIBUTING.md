## How to Contribute

### Reporting Bugs
- Check if the issue already exists
- Include your Notion database schema
- Share error messages (remove personal tokens!)
- Describe what you expected vs. what happened

### Suggesting Features
- Explain your use case
- Describe the feature you'd like
- Consider if it would be useful to others

### Submitting Code
1. Fork the repository
2. Create a feature branch: `git checkout -b feature-name`
3. Make your changes
4. Test thoroughly
5. Commit: `git commit -m "Add feature-name"`
6. Push: `git push origin feature-name`
7. Open a Pull Request

### Code Style
- Follow PEP 8 for Python code
- Add comments for complex logic
- Keep functions small and focused
- Test with different Notion database configurations

### Testing
Before submitting:
- Test with a real Readwise account
- Test with a real Notion database
- Verify no tokens are hardcoded
- Check that GitHub Actions workflow still works

## Questions?
Open an issue and we'll help!
